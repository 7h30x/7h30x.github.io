    This week started on the Monday, when I first stepped in for the Hacktiv8 introduction session in the morning. There we were introduced to the general scope and structure of the program and what the expectations were regarding time commitments.     On a positive note, the difficulty of packing in so much information into the brain forced me to structure my time and pick what was really important to learn. The most difficult of all the content was not the content material itself but picking the right resources to learn from and the sheer amount of knowledge available.     Lessons From Installing Dual-Boot Ubuntu onto Existing Windows 10

    Post caption

    Step 1 is download Linux distribution that is the right one for your PC (64-bit) and…this is important…create file-level backup of the current OS disk drive!

    Step 2 is creating a bootable Disk Image (.iso), meaning bootable on machine’s BIOS/UEFI system using a tool like Rufus. Make sure partition format of bootable is same as target disk.

    Step 3 is creating Disk Partition for the new OS through disk utility tools; this should be around 100GB.

    Step 4 is installing the OS from the bootable.

    • Make sure that PC boot mode is UEFI- not legacy BIOS mode, otherwise the install will cause errors down the line…
    Installing 2 OS in different boot mode will cause complications every time you want to choose OS from boot screen.

    In the case of errors in getting to recognize bootloader USB/DVD drives, correct BIOS/UEFI setup is important—within Windows 10,
    the menu path to setup is:

    • PC Settings> Update & Security> Recovery> Restart Now> Advanced Options> UEFI settings
    • Fast boot should be OFF, as it may cause unloaded USB drivers (making USB- bootable unable to be recognized)
    • USB-Enabled Boot should be ON and PRIORITY
    • Legacy Boot should be OFF (this turns of UEFI and uses BIOS to boot)
    • SATA connection setting on UEFI setup should be ADHC (allows bootable to read hard drive for installation of Ubuntu)
